# Sprint 003 — Cabinet Dashboard

> **Статус:** Planning
> **Старт:** 2026-02-25
> **Владелец:** Иван

---

## Цель спринта

Создать таб **"Кабинет"** — единый профиль клиента (магазина) с полной информацией для менеджера R5.

Спринт фокусируется на том, что **можно безопасно реализовать сейчас** — на существующих данных и простых CRM-полях. Сложные подсистемы (биллинг, учёт удалений, отчётность) выносятся в отдельное планирование.

---

## Принцип: сначала безопасное, потом сложное

| Категория | Статус | Когда |
|---|---|---|
| **Кабинет MVP** (существующие данные) | В спринте | Phase 1 — сейчас |
| **CRM** (контакты, ссылки, заметки) | В спринте | Phase 2 — сразу после |
| **Учёт удалений** | Отдельное планирование | Требует исследования |
| **Отчётность** | Отдельное планирование | После учёта удалений |
| **Биллинг** | Отдельное планирование | После отчётности |

## Отложено — требует отдельного планирования

### Учёт удалений (будущий Sprint)

> **Почему отдельно:** Это не просто "посчитать". Нужна система определения, КТО удалил отзыв.

Два источника удалений:
1. **R5-жалобы** — `review_complaints.status = 'approved'` (уже отслеживаем)
2. **Чат-удаления** — клиент сам удалил после диалога. Определяется по триггерным сообщениям в чате (напр. "ваша заявка на кешбек..."). Нужно:
   - Проанализировать диалоги, найти паттерны триггерных сообщений
   - Продумать систему детекции и учёта
   - Связать с `reviews.deleted_from_wb_at`

**Зависимости:** Отчётность и биллинг не могут работать корректно без этой системы.

### Отчётность (будущий Sprint)

> **Почему отдельно:** Зависит от учёта удалений. Без него отчёт неполный.

Планируемый функционал:
- Еженедельные отчёты для клиентов (автогенерация)
- Интеграция с TG Mini App (отчёт прямо в Telegram)
- Интеграция с Google Sheets (текущие таблицы)
- Лента событий в кабинете

Задачи TASK-008..010 остаются как черновики, будут детализированы при планировании.

### Биллинг (будущий Sprint)

> **Почему отдельно:** Зависит от учёта удалений + отчётности. Банк: Точка.

Планируемый функционал:
- Цена за удалённый отзыв
- Автоматический расчёт суммы за период
- Выставление счетов
- API Точка Банк (входящие платежи, сверка)
- Учёт оплат и задолженности

Задачи TASK-005..007 остаются как черновики, будут пересмотрены.
