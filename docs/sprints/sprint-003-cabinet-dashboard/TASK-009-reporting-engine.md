## Goal

Автоматическая генерация еженедельных отчётов для клиентов. Шаблон HTML → snapshot метрик → PDF.

## Scope

### Шаблон отчёта (содержание)

```
📊 Еженедельный отчёт R5
Магазин: {name} | Период: {start} — {end}

─── Отзывы ───
Новых отзывов: +N
  1★: X | 2★: X | 3★: X | 4★: X | 5★: X

─── Жалобы ───
Подано: N | Одобрено: N | Отклонено: N
Процент одобрения: XX%

─── Удаления ───
Удалено с WB за период: N
Накопительно: N (всего)

─── Чаты ───
Новых диалогов: N
Рейтинг повышен (→5★): N
Авто-цепочки завершены: N

─── Финансы ───
Удалено: N × {price} = {total} руб
```

### Cron
- `startWeeklyReportGenerator` — каждый понедельник в 09:00 MSK
- Для всех active stores генерирует отчёт за прошлую неделю (пн-вс)
- Сохраняет в `store_reports.data` как JSONB

### API
```
GET    /api/stores/[storeId]/reports              — список отчётов
POST   /api/stores/[storeId]/reports/generate     — ручная генерация
GET    /api/stores/[storeId]/reports/[id]         — конкретный отчёт
```

### UI
- Секция "Отчёты" в кабинете: список с датами
- Клик → модал с отчётом (HTML rendered)
- Кнопка "Сгенерировать" для ручного запуска

## Estimated Effort
- ~5-6 часов (шаблон + cron + API + UI)
