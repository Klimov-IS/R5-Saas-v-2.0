# A/B Testing: Шаблоны жалоб для пустых отзывов

**Статус:** ✅ РЕАЛИЗОВАНО
**Дата:** 20 января 2026

---

## Обзор

Для **пустых отзывов (1-3 звезды без текста, pros, cons)** система автоматически использует **шаблоны** вместо AI-генерации, экономя токены и затраты.

Теперь доступны **4 варианта шаблонов** с **автоматической ротацией** для A/B тестирования эффективности разных формулировок.

## Изменения

### 1. Расширенный критерий использования шаблонов

**Старое правило:**
- Пустые отзывы с рейтингом **1-2 звезды** → шаблон

**Новое правило:**
- Пустые отзывы с рейтингом **1-3 звезды** → шаблон

**Эффект:** Больше отзывов получат бесплатные шаблоны вместо AI-генерации

### 2. Четыре варианта шаблонов

Все шаблоны используют **официальные категории WB API** (reason_id 11-20):

#### Вариант A — reason_id: 11 (Отзыв не относится к товару)
**Фокус:** Отсутствие информации о товаре

```
Данный отзыв не относится к товару, так как не содержит никакой информации..."
```

#### Вариант B — reason_id: 12 (Отзыв оставили конкуренты)
**Фокус:** Необоснованная низкая оценка (возможные действия конкурентов)
```
"Подобные отзывы с необоснованными низкими оценками без текста характерны
для действий недобросовестных конкурентов..."
```

#### Вариант C — reason_id: 20 (Другое)
**Фокус:** Нарушение стандартов платформы
```

Отзыв не соответствует стандартам качества платформы..."
```

#### Вариант D — reason_id: 11 (Отзыв не относится к товару) — Альтернативная формулировка
**Фокус:** Затруднение выбора для других покупателей
```

```

## Механизм ротации

### Hash-based Distribution

Каждый `reviewId` хешируется, и результат определяет вариант шаблона:

```typescript
function simpleHash(str: string): number {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    hash = ((hash << 5) - hash) + str.charCodeAt(i);
    hash = hash & hash;
  }
  return Math.abs(hash);
}

// Variant selection
const variantIndex = hash(reviewId) % 4; // 0-3
```

### Преимущества

✅ **Консистентность** — один и тот же отзыв всегда получает один шаблон
✅ **Равномерность** — ~25% на каждый вариант (протестировано на 100 отзывах)
✅ **Детерминированность** — нет random, предсказуемо

## Логирование

При использовании шаблона выводится лог:

```
[TEMPLATE] Using template for empty review (zero AI cost)
[TEMPLATE] Review abc123: rating=2, variant=B (1), reason=12, text=empty, pros=empty, cons=empty
```

**Формат:**
- `rating` — оценка отзыва (1-3)
- `variant` — буква варианта (A/B/C/D)
- `(number)` — variantId (0-3)
- `reason` — reason_id WB категории

## A/B Аналитика (будущее)

### Цель

Определить, какая формулировка имеет **наибольший approval_rate** у модерации WB.

### План

1. Собрать данные по 1000+ жалоб на каждый вариант
2. Сравнить approval_rate:
   ```sql
   SELECT
     template_variant,
     COUNT(*) as total,
     COUNT(CASE WHEN status = 'approved' THEN 1 END) as approved,
     ROUND(COUNT(CASE WHEN status = 'approved' THEN 1 END)::float /
           NULLIF(COUNT(CASE WHEN status IN ('approved','rejected') THEN 1 END), 0) * 100, 2) as approval_rate
   FROM review_complaints
   WHERE ai_cost_usd = 0 -- only templates
   GROUP BY template_variant
   ORDER BY approval_rate DESC;
   ```
3. Если один вариант показывает +10-15% лучший approval_rate → переключиться только на него

### Добавить поле в БД (опционально)

Для более точной аналитики можно добавить колонку:

```sql
ALTER TABLE review_complaints
ADD COLUMN template_variant_id INT NULL;
```

И сохранять `template.variantId` при создании жалобы.

## Как использовать

Система работает **автоматически**:

1. При генерации жалобы проверяется условие: `rating 1-3 && пустой текст`
2. Если условие выполнено → выбирается шаблон по хешу `reviewId`
3. Возвращается результат с `cost_usd = 0`
4. В логах отображается вариант (A/B/C/D)

**Никаких дополнительных действий не требуется.**

## Мониторинг

### Проверка использования шаблонов

```sql
-- Количество жалоб по шаблонам vs AI
SELECT
  CASE
    WHEN ai_cost_usd = 0 THEN 'Template (zero cost)'
    ELSE 'AI generated'
  END as generation_type,
  COUNT(*) as count
FROM review_complaints
GROUP BY generation_type;
```

```sql
-- Топ категорий жалоб
SELECT
  reason_id,
  reason_name,
  COUNT(*) as count
FROM review_complaints
WHERE ai_cost_usd = 0
GROUP BY reason_id, reason_name
ORDER BY count DESC;
```

## FAQ

### Можно ли вручную выбрать вариант шаблона?

Нет, система автоматически выбирает вариант по хешу `reviewId` для консистентности.

### Что если один вариант показывает очень плохой approval_rate?

Можно удалить этот вариант из `TEMPLATE_VARIANTS` массива в `complaint-templates.ts` и изменить модуль на `hash % 3` вместо `hash % 4`.

### Почему не используем random вместо hash?

Random может давать разные результаты при повторных запросах. Hash гарантирует, что один и тот же отзыв всегда получит один и тот же шаблон.
