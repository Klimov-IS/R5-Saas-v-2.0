# Политика работы с OZON

**Дата:** 2026-03-04
**Версия:** 1.0
**Назначение:** Обязательные правила работы с OZON. Весь код должен соответствовать этим правилам.

---

## 1. Принципиальные отличия от WB

| Параметр | WB | OZON |
|----------|-----|------|
| Жалобы на отзывы | Да (1 раз) | **НЕТ** — механизм не существует |
| Отправка сообщений | Без ограничений (с reply_sign) | **1 сообщение → ждём ответа** покупателя |
| Подписка | Не требуется | **Premium Plus** обязателен для чатов |
| Rate limit | ~5-10 req/min | **50 req/sec** |
| API формат | GET + POST | **Все POST** с JSON |
| Регистр перечислений | Свободный | **Строго UPPERCASE** (BUYER_SELLER, OPENED) |

---

## 2. КРИТИЧЕСКОЕ ПРАВИЛО: 1 сообщение до ответа

OZON ограничивает продавца:
- После отправки первого сообщения покупателю, **отправка заблокирована** до ответа покупателя
- OZON API вернёт **error code 7** ("chat not started by buyer") при попытке отправить второе сообщение
- Это НЕ баг — это архитектурное ограничение OZON

### Последствия для системы
- Рассылки на OZON: первое сообщение отправляется, затем рассылка ждёт в `awaiting_reply`
- Если покупатель НЕ ответит, последующие сообщения рассылки будут возвращать error code 7
- Система должна корректно обработать этот error (permanent stop, а не бесконечный retry)

---

## 3. Жалобы

**OZON не поддерживает жалобы на отзывы.** Нет API, нет механизма, нет кнопки.

Работа с негативными отзывами на OZON ведётся **исключительно через чаты** с покупателями.

- `isReviewResolvedForChat()` не проверяется для OZON чатов при автозакрытии (нет жалоб)
- Статус `complaint_status` для OZON отзывов всегда `NULL` или `'not_sent'`

## 4. Диалоги (чаты)

### Два типа OZON чатов

| Тип | Кто инициирует | `product_nm_id` | Объём | Обрабатывается R5? |
|-----|---------------|-----------------|-------|---------------------|
| **Seller-initiated** | Продавец из карточки отзыва | Заполнен (`context.sku`) | ~481 | **Да** |
| **Buyer-initiated** | Покупатель | NULL | ~317K | **Нет** |

**R5 работает ТОЛЬКО с seller-initiated чатами.**

### Как создаётся seller-initiated чат
1. Продавец находит согласованный отзыв в R5
2. Открывает чат с покупателем на OZON (через OZON dashboard)
3. OZON автоматически добавляет `context.sku` в первое сообщение
4. R5 sync подхватывает чат, извлекает `context.sku` → записывает `product_nm_id`

### Синхронизация
- **Tier 1**: Unread-only scan каждые 5/15/60 мин (адаптивно)
- **Tier 2**: Full scan каждый час 9:00-20:00 МСК
- Инкрементальный skip по `ozon_last_message_id`

---

## 5. Статусы чатов

Те же 4 статуса что и для WB: `awaiting_reply`, `inbox`, `in_progress`, `closed`.

### Переходы (те же правила что и WB)
- Покупатель ответил → `inbox`
- Продавец ответил → `in_progress` (с защитой awaiting_reply при активной рассылке)
- Рассылка запущена → `awaiting_reply`
- Закрытие → `closed` + `completion_reason`

## 6. Работа с отзывами на OZON — Дополнение (не удаление!)

### КРИТИЧЕСКОЕ ОТЛИЧИЕ от WB
На WB цель — **удаление** негативного отзыва (через жалобу или через чат с покупателем).
На OZON цель — **дополнение** отзыва покупателем с изменением оценки на 5★.

| Параметр | WB | OZON |
|----------|-----|------|
| Удаление отзыва | Да (через жалобу или чат) | **НЕТ** — отзывы не удаляются |
| Изменение отзыва | Нет | **Да** — покупатель может редактировать |
| Цель работы | Удалить негативный отзыв | Покупатель ставит 5★ |
| Компенсация | Кешбэк/возврат за удаление | Кешбэк/возврат + просьба обновить до 5★ |

### Инструкция для покупателя (отправляется в фазе "решение")

> **Как дополнить отзыв на OZON и поставить 5★:**
> 1. Откройте приложение Ozon
> 2. Нажмите иконку профиля внизу справа
> 3. Перейдите в раздел «Отзывы» → «Мои отзывы»
> 4. Найдите нужный товар и откройте свой отзыв
> 5. Нажмите «Редактировать»
> 6. Поставьте оценку 5★
> 7. Нажмите «Отправить отзыв»

### AI правила для OZON чатов
- **НЕЛЬЗЯ:** упоминать "удаление отзыва", "убрать отзыв", "снять отзыв"
- **МОЖНО:** "дополнить отзыв", "обновить оценку", "пересмотреть оценку", "изменить отзыв"
- Инструкция отправляется **ТОЛЬКО после согласия** покупателя (фаза "решение")
- В фазах "знакомство" и "понимание" — стандартная работа (узнай проблему, прояви эмпатию)
- Реализовано в: `src/lib/ai-context.ts` → `OZON_MARKETPLACE_ADDENDUM`

## 7. Рассылки на OZON

### Особенности
- Первое сообщение отправляется (OZON разрешает)
- Последующие сообщения: OZON заблокирует, пока покупатель не ответит
- Error code 7 → рассылка останавливается автоматически (`stop_reason = 'send_failed'`)
- Шаблоны рассылок: отдельные OZON-версии (`DEFAULT_OZON_FOLLOWUP_TEMPLATES_*`)

### Что НЕ работает
- Массовая рассылка нескольких сообщений подряд (OZON заблокирует)
- Retry при error code 7 (бесполезно — покупатель не ответил)

## 8. Фильтрация очереди TG

Для OZON чатов в TG очереди:
- Фильтр: `c.marketplace = 'ozon' AND c.product_nm_id IS NOT NULL`
- Это показывает только seller-initiated чаты
- product_rules.work_in_chats **не обязателен** для OZON (NULL допускается)

## 9. API

### Аутентификация
- `Client-Id` (числовой) + `Api-Key` (UUID) в headers
- Base URL: `https://api-seller.ozon.ru`
- Все запросы **POST** с JSON body

### Основные методы (OzonApiClient)
| Метод | Назначение |
|-------|-----------|
| `getChatList()` | Список чатов (BUYER_SELLER, status OPENED) |
| `getChatHistory()` | История сообщений чата |
| `sendChatMessage()` | Отправка сообщения (ограничение: 1 до ответа) |
| `getProductList()` | Список товаров |
| `getReviewList()` | Список отзывов (Premium Plus) |

### Ограничения подписки
| Подписка | Чаты | Отзывы |
|----------|------|--------|
| PREMIUM | Да | 403 Forbidden |
| PREMIUM PLUS | Да | Да |

## 10. Данные и ID

### Формат ID товаров
- `{storeId}_ozon_{ozon_product_id}` — внутренний формат R5
- Тройной ID: `product_id` / `offer_id` / `sku`
- SKU sources: `sds` = FBO, `fbs` = FBS

### Matching товаров
- OZON чат → `product_nm_id` (это ozon_sku из context.sku)
- Matching: `c.product_nm_id = p.ozon_sku OR c.product_nm_id = p.ozon_fbs_sku`
